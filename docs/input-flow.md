# Input Flow

## 目的

Flow の入力体験を「構造化してから書く」ではなく、「まず記録してから整理する」順序に変える。

## 画面フロー

### 1段階目: キャプチャ

- 表示要素はテキストボックス、写真追加、カメラ起動、`次へ`
- この段階ではカテゴリもタグも表示しない
- 下書きは `sessionStorage` に保持する
- `次へ` を押すまでは Supabase に保存しない

### 2段階目: 確認シート

- ボトムシートで開く
- 入力内容のプレビューを表示する
- カテゴリは大きなタップターゲットで選択する
- デフォルトカテゴリは `event`
- タグはチップ型の単一選択 UI を使い、未選択のまま保存できる
- `task` を選んだ場合のみ優先度と一括登録を表示する
- `schedule` を選んだ場合のみ日時入力を表示する
- `戻る` で 1 段階目へ戻り、内容を編集できる

## 保存仕様

- 最終保存時に既存の `addBlockWithDate` を呼ぶ
- `blocks.category` と `blocks.tag` の Supabase スキーマ変更は不要
- 1段階目で未分類保存はしない

## 確認観点

- 1段階目では保存が発生しないこと
- 写真のみ投稿でも 2段階目へ進めること
- 何も変更しなければ `event` として保存されること
- タグはドロップダウンなしで選択・解除できること
- `task` と `schedule` の追加入力が 2 段階目に閉じていること
